# 家庭菜品管理与智能食谱小程序

## 项目简介

这是一个基于微信小程序的家庭菜品管理与智能食谱推荐系统。

## 功能特性

- 🥬 菜品管理：添加、编辑、删除菜品，支持分类管理
- 📸 智能识别：拍照识别菜品，自动填充信息
- 🍳 智能食谱：根据现有菜品生成推荐食谱
- 👥 社区分享：发布和浏览食谱，点赞收藏
- 📊 健康报告：查看每日/每周营养摄入情况
- ⏰ 过期提醒：自动提醒即将过期的菜品

## 技术栈

- 微信小程序原生开发
- 微信云开发（云函数、云数据库、云存储）
- Node.js（云函数）

## 项目结构

```
WeChatTool_WhatToEat/
├── WhatToEat/              # 小程序主目录
│   ├── pages/             # 页面文件
│   ├── components/        # 组件
│   ├── cloudfunctions/    # 云函数
│   ├── utils/             # 工具函数
│   └── config/            # 配置文件
└── README.md
```

## 环境配置

1. 复制 `.env.example` 为 `.env`
2. 填写云开发环境ID和相关配置

## 开发指南

### 本地开发

1. 使用微信开发者工具打开 `WhatToEat` 目录
2. 配置云开发环境
3. 上传并部署云函数

### 云函数部署

在微信开发者工具中，右键每个云函数文件夹，选择"上传并部署：云端安装依赖"

## 团队协作

### Git 工作流

1. 创建功能分支：`git checkout -b feature/your-feature`
2. 提交更改：`git commit -m "描述你的更改"`
3. 推送到远程：`git push origin feature/your-feature`
4. 创建 Pull Request 进行代码审查

### 提交规范

- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具相关

## 注意事项

- 不要提交 `.env` 文件（包含敏感信息）
- 不要提交 `project.private.config.json`
- 提交前确保代码可以正常运行
